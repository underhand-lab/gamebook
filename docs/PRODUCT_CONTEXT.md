# Product Context

이 프로젝트는 야구 경기 단위 Letterboxd + 마이크로블로그 스타일의 리뷰 앱이다.

사용자는 당일 기록이나 시청 인증 없이, 과거 또는 현재의 야구 경기를 찾아 짧은 리뷰를 남길 수 있다.

현재 MVP에서는 리뷰 대상이 오직 “경기”로 한정된다.

## 지원하는 것

현재 MVP에서 사용자는 다음을 할 수 있어야 한다.

- 야구 경기를 찾거나 선택한다.
- 선택한 경기에 대해 리뷰를 작성한다.
- 별점을 남긴다. 단, 별점은 선택 사항이어야 한다.
- 짧은 감상을 남긴다.
- 감정 태그를 붙인다.
- 관람 방식을 선택한다.
- 공개 범위를 선택한다.
- 나중에 Fediverse로 공개될 수 있도록 Fediverse 공개 여부를 선택한다.

## 지원하지 않는 것

현재 MVP에서는 다음을 지원하지 않는다.

- 선수 리뷰
- 팀 리뷰
- 구장 리뷰
- 시즌 리뷰
- 특정 장면 리뷰
- 응원가 리뷰
- 유니폼 리뷰
- 자유 야구 감상 포스트
- 경기와 무관한 마이크로블로그 포스트
- targetType 선택 기능
- 자유 포스트 작성 기능

## 제품 지향

이 앱은 단순한 스코어 기록 앱이 아니다.

핵심은 “경기를 봤는지 인증하는 것”이 아니라, 사용자가 특정 야구 경기를 어떻게 기억하고 해석하는지를 기록하는 것이다.

예시 리뷰:

- “이 경기는 스코어보다 9회말 공기가 오래 남을 것 같다.”
- “졌는데 이상하게 괜찮았다.”
- “이긴 경기인데 다시 보고 싶지는 않다.”
- “하이라이트만 봤는데도 감정 소모가 심했다.”
- “직관 갔으면 평생 기억했을 경기.”

## 중요한 제품 원칙

1. 시청 인증을 요구하지 않는다.
2. 당일 기록만 허용하지 않는다.
3. 과거 경기에도 리뷰를 남길 수 있어야 한다.
4. 별점은 선택 사항이어야 한다.
5. 짧은 마이크로블로그형 감상이 자연스러워야 한다.
6. 감정 태그를 중요하게 본다.
7. 현재는 경기 리뷰만 지원한다.
8. 나중에 ActivityPub federation을 붙일 수 있도록 설계한다.
9. 하지만 현재 프론트 작업에서는 실제 ActivityPub 서버 구현을 하지 않는다.

## ActivityPub 방향

이 앱은 나중에 Fediverse와 연결될 수 있다.

앱 내부에서는 리뷰가 구조화된 경기 리뷰로 보인다.

예시:

- 경기: 2026-07-01 LG vs 두산
- 별점: 4.5
- 감정 태그: 낭만, 끝내기, 잠실
- 관람 방식: TV
- 리뷰: “이 경기는 스코어보다 9회말의 공기가 오래 남을 것 같다.”

앱 외부에서는 일반 Fediverse 포스트처럼 보일 수 있다.

예시:

“★★★★½ 2026-07-01 LG vs 두산 리뷰. 이 경기는 스코어보다 9회말의 공기가 오래 남을 것 같다. #KBO #야구리뷰”

## 현재 범위

현재 범위는 프론트엔드 정리다.

해야 할 것:

- MockAPI 호출부 분리
- 환경변수 기반 API base URL 사용
- 경기 리뷰 중심의 타입 정리
- ActivityPub-ready 타입 필드 추가
- 공개 범위 UI 준비
- Fediverse 공개 여부 UI 준비
- canonical URL / federation status 표시 준비

하지 말 것:

- 실제 ActivityPub 서버 구현
- WebFinger 구현
- HTTP Signature 구현
- inbox/outbox delivery 구현
- remote actor fetch 구현
- remote follow 구현
- remote reply 수신 구현
- remote like 수신 구현
- GitHub Actions 설정
- Vercel 배포 설정
- Spring Boot 백엔드 생성
